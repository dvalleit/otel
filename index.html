Module 01 · Foundations
Módulo 01 · Fundamentos

What is OpenTelemetry?

¿Qué es OpenTelemetry?

OpenTelemetry (OTel) is a CNCF project that provides a vendor-neutral, open-standard framework for collecting traces, metrics, and logs from your applications — and routing them to any backend.

OpenTelemetry (OTel) es un proyecto CNCF que provee un framework de estándar abierto y neutral para recolectar trazas, métricas y logs de tus aplicaciones, y enviarlos a cualquier backend.

🔗
Traces
Trazas

Track the journey of a request across microservices. Made of spans — each representing a unit of work.

Seguimiento del recorrido de una solicitud a través de microservicios. Compuestas de spans, cada uno representando una unidad de trabajo.

📊
Metrics
Métricas

Numerical measurements over time: counters, gauges, and histograms. Ideal for dashboards and alerts.

Mediciones numéricas en el tiempo: contadores, indicadores e histogramas. Ideales para dashboards y alertas.

📋
Logs
Logs

Timestamped records of events. OTel correlates logs with traces and metrics automatically.

Registros de eventos con marca de tiempo. OTel correlaciona logs con trazas y métricas automáticamente.

🏗️
Architecture
Arquitectura

API → SDK → Collector → Backend. The Collector acts as a vendor-agnostic pipeline.

API → SDK → Colector → Backend. El Colector funciona como pipeline agnóstico del proveedor.

Your First Trace

Tu Primera Traza

Install the SDK and emit your first span to the console in under 5 minutes.

Instala el SDK y emite tu primer span a la consola en menos de 5 minutos.

Install dependencies:

Instalar dependencias:

  @opentelemetry/sdk-trace-base \
  @opentelemetry/resources \
  @opentelemetry/semantic-conventions

Create your first span:

Crear tu primer span:

from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import (
    BatchSpanProcessor, ConsoleSpanExporter
)

# 1. Set up the provider
provider = TracerProvider()
provider.add_span_processor(
    BatchSpanProcessor(ConsoleSpanExporter())
)
trace.set_tracer_provider(provider)

# 2. Get a tracer
tracer = trace.get_tracer(__name__)

# 3. Create a span
with tracer.start_as_current_span("my-first-span") as span:
    span.set_attribute("user.name", "OTel Hero")
    span.set_attribute("operation.type", "greeting")
    print("Hello, OpenTelemetry! 🎉")
const { NodeSDK } = require('@opentelemetry/sdk-node');
const { ConsoleSpanExporter } = require('@opentelemetry/sdk-trace-base');
const { resourceFromAttributes } = require('@opentelemetry/resources');
const { ATTR_SERVICE_NAME } = require('@opentelemetry/semantic-conventions');
const { trace } = require('@opentelemetry/api');

// 1. Start the SDK
const sdk = new NodeSDK({
  resource: resourceFromAttributes({
    [ATTR_SERVICE_NAME]: 'my-first-service',
  }),
  traceExporter: new ConsoleSpanExporter(),
});
sdk.start();

// 2. Get a tracer and create a span
const tracer = trace.getTracer('my-tracer');
const span = tracer.startSpan('my-first-span');
span.setAttribute('user.name', 'OTel Hero');
span.setAttribute('operation.type', 'greeting');

console.log('Hello, OpenTelemetry! 🎉');
span.end();
await sdk.shutdown();

# Counter: track how many requests we handle

    "http.request.duration",
    unit="ms",

request_counter.add(1, {"method": "GET", "route": "/api/users"})

Context Propagation & Sampling

Propagación de Contexto y Muestreo

Propagate trace context across service boundaries and control telemetry volume with sampling strategies.

Propaga el contexto de traza entre servicios y controla el volumen de telemetría con estrategias de muestreo.

import requests
from opentelemetry import trace
from opentelemetry.propagate import inject, extract
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter

# OTLP exporter — sends to the Collector
provider = TracerProvider()
provider.add_span_processor(
    BatchSpanProcessor(OTLPSpanExporter(endpoint="http://localhost:4317"))
)
trace.set_tracer_provider(provider)
tracer = trace.get_tracer(__name__)

# SERVICE A — injects context into outgoing headers
def call_service_b():
    with tracer.start_as_current_span("service-a: call-b") as span:
        headers = {}
        inject(headers)           # <-- injects traceparent header
        span.set_attribute("http.url", "http://service-b/data")
        response = requests.get("http://service-b/data", headers=headers)
        span.set_attribute("http.status_code", response.status_code)
        return response.json()
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.sampling import (
    ParentBased,
    TraceIdRatioBased,
    ALWAYS_ON,
    ALWAYS_OFF,
)

# Sample 10% of root traces; respect parent decision for child spans
sampler = ParentBased(
    root=TraceIdRatioBased(0.1),           # 10% sampling rate
    remote_parent_sampled=ALWAYS_ON,       # always sample if parent says so
    remote_parent_not_sampled=ALWAYS_OFF,  # drop if parent dropped
)

provider = TracerProvider(sampler=sampler)

# For tail-based sampling, configure it in the OTel Collector:
# processors:

#     policies:
#       - name: errors-policy
#         type: status_code
#         status_code: {status_codes: [ERROR]}
#       - name: slow-traces-policy
#         type: latency
#         latency: {threshold_ms: 500}

Custom Attributes, Events & Errors
Atributos, Eventos y Errores Personalizados
Intermediate  ~20 min
▶
Objective: Enrich spans with semantic attributes, add events (logs on a span), and correctly record errors using span.record_exception().
Objetivo: Enriquecer spans con atributos semánticos, añadir eventos (logs en un span) y registrar errores correctamente usando span.record_exception().
from opentelemetry import trace
from opentelemetry.trace import StatusCode
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter

provider = TracerProvider()
provider.add_span_processor(BatchSpanProcessor(ConsoleSpanExporter()))
trace.set_tracer_provider(provider)
tracer = trace.get_tracer(__name__)

def process_order(order_id: str, amount: float):
    with tracer.start_as_current_span("process-order") as span:
        # Semantic attributes (follow OTel conventions)
        span.set_attribute("order.id", order_id)
        span.set_attribute("order.amount", amount)
        span.set_attribute("order.currency", "USD")

        # Add an event (a timestamped log on the span)
        span.add_event("Order validation started")

        try:
            if amount <= 0:
                raise ValueError(f"Invalid amount: {amount}")

            span.add_event("Payment processed", {"payment.method": "card"})
            span.set_status(StatusCode.OK)
            return {"status": "success"}

        except Exception as e:
            # Record the exception and mark span as error
            span.record_exception(e)
            span.set_status(StatusCode.ERROR, str(e))
            return {"status": "error", "message": str(e)}

process_order("ORD-001", 99.99)
process_order("ORD-002", -5)    # triggers error path
const { trace, SpanStatusCode } = require('@opentelemetry/api');
const { NodeSDK } = require('@opentelemetry/sdk-node');
const { ConsoleSpanExporter } = require('@opentelemetry/sdk-trace-base');

const sdk = new NodeSDK({ traceExporter: new ConsoleSpanExporter() });
sdk.start();

const tracer = trace.getTracer('exercise-3');

async function processOrder(orderId, amount) {
  const span = tracer.startSpan('process-order');
  span.setAttribute('order.id', orderId);
  span.setAttribute('order.amount', amount);
  span.setAttribute('order.currency', 'USD');
  span.addEvent('Order validation started');

  try {
    if (amount <= 0) throw new Error(`Invalid amount: ${amount}`);
    span.addEvent('Payment processed', { 'payment.method': 'card' });
    span.setStatus({ code: SpanStatusCode.OK });
    return { status: 'success' };
  } catch (err) {
    span.recordException(err);
    span.setStatus({ code: SpanStatusCode.ERROR, message: err.message });
    return { status: 'error', message: err.message };
  } finally {
    span.end();
  }
}

processOrder('ORD-001', 99.99);
processOrder('ORD-002', -5);

Distributed Tracing Across Two Services
Trazas Distribuidas Entre Dos Servicios
Advanced  ~30 min
▶
Objective: Propagate trace context from Service A to Service B over HTTP so both appear as a single connected trace in Jaeger.
Objetivo: Propagar el contexto de traza desde el Servicio A al Servicio B vía HTTP para que ambos aparezcan como una sola traza conectada en Jaeger.
# service_a.py — calls Service B, injects trace context
import requests
from opentelemetry import trace
from opentelemetry.propagate import inject
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter
from opentelemetry.instrumentation.requests import RequestsInstrumentor

provider = TracerProvider()
provider.add_span_processor(BatchSpanProcessor(ConsoleSpanExporter()))
trace.set_tracer_provider(provider)

# Auto-instrument the requests library to inject headers automatically
RequestsInstrumentor().instrument()

tracer = trace.get_tracer("service-a")

with tracer.start_as_current_span("service-a: handle-request") as span:
    span.set_attribute("service.name", "service-a")
    # Headers injected automatically by RequestsInstrumentor
    response = requests.get("http://localhost:5001/data")
    print("Service B response:", response.json())

import logging

    else:
        logger.warning("Anonymous player is rolling the dice: %s", result)
    return result

Decorador de Span, Links y Métricas Asíncronas (Docs Oficiales de Python)
Intermediate  ~20 min  ·  opentelemetry.io ↗
▶
Objective: Use the @tracer.start_as_current_span decorator to instrument functions without boilerplate, create Span Links between independent operations, and set up an async Observable Gauge for external metric sources.
Objetivo: Usar el decorador @tracer.start_as_current_span para instrumentar funciones sin boilerplate, crear Span Links entre operaciones independientes, y configurar un Observable Gauge asíncrono para fuentes de métricas externas.
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter

provider = TracerProvider()
provider.add_span_processor(BatchSpanProcessor(ConsoleSpanExporter()))
trace.set_tracer_provider(provider)

# The tracer must be module-level for the decorator to work
tracer = trace.get_tracer("exercise-8")

# Using the decorator is equivalent to wrapping the body in
# with tracer.start_as_current_span("do_work"): ...
@tracer.start_as_current_span("validate-input")
def validate_input(data: dict) -> bool:
    # The span is automatically started and closed around this function
    span = trace.get_current_span()
    span.set_attribute("input.keys", list(data.keys()))
    return all(v is not None for v in data.values())

@tracer.start_as_current_span("process-data")
def process_data(data: dict):
    span = trace.get_current_span()
    if validate_input(data):        # validate-input span is a CHILD of process-data
        span.set_attribute("validation.passed", True)
        return {"result": "processed"}
    raise ValueError("Invalid input")

process_data({"user": "Alice", "amount": 42})

# Span Links: causally associate two independent traces
# Use case: an async job triggered by an HTTP request

# Step 1: original request creates a span
with tracer.start_as_current_span("http-request") as request_span:
    # Capture the context to link from the async job
    request_ctx = request_span.get_span_context()
    link_to_request = trace.Link(request_ctx)
    print(f"Request span ID: {hex(request_ctx.span_id)}")

# Step 2: async job runs in a DIFFERENT trace but links back
with tracer.start_as_current_span("async-job", links=[link_to_request]) as job_span:
    # job_span is NOT a child of request_span — it's in its own trace
    # but it carries a Link pointing to the request span
    job_span.set_attribute("job.type", "email-notification")
    print("Async job executed, linked to original request")

from typing import Iterable
import random
from opentelemetry import metrics
from opentelemetry.metrics import CallbackOptions, Observation
from opentelemetry.sdk.metrics import MeterProvider
from opentelemetry.sdk.metrics.export import (
    ConsoleMetricExporter, PeriodicExportingMetricReader
)

reader = PeriodicExportingMetricReader(
    ConsoleMetricExporter(), export_interval_millis=5000
)
provider = MeterProvider(metric_readers=[reader])
metrics.set_meter_provider(provider)
meter = metrics.get_meter("exercise-8-gauge")

# Callback: called by the SDK every export interval
def cpu_usage_callback(options: CallbackOptions) -> Iterable[Observation]:
    # In production this would read from /proc/stat or psutil
    for core_id in range(4):
        usage = random.uniform(0.1, 0.95)
        yield Observation(usage, {"cpu.core": str(core_id)})

meter.create_observable_gauge(
    "system.cpu.utilization",
    callbacks=[cpu_usage_callback],
    unit="1",
    description="CPU utilization per core (0.0–1.0)",
)

import time
print("Collecting metrics for 10 seconds...")
time.sleep(10)   # reader fires at 5s and 10s

OTel Logging — Correlate Logs with Traces (Official Python Docs)
Logging con OTel — Correlacionar Logs con Trazas (Docs Oficiales Python)
Intermediate  ~20 min  ·  opentelemetry.io ↗
▶
Objective: Wire Python's built-in logging module into OTel's Log SDK so every log record is automatically stamped with the current trace_id and span_id — enabling log-to-trace correlation in any observability backend.
Objetivo: Conectar el módulo logging de Python con el SDK de Logs de OTel para que cada registro de log sea automáticamente marcado con el trace_id y span_id actuales — habilitando correlación log-a-traza en cualquier backend.
# Install OTel logging SDK
pip install opentelemetry-sdk opentelemetry-api
import logging
from opentelemetry import trace
from opentelemetry._logs import set_logger_provider
from opentelemetry.sdk._logs import LoggerProvider, LoggingHandler
from opentelemetry.sdk._logs.export import (
    BatchLogRecordProcessor, ConsoleLogRecordExporter
)
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor, ConsoleSpanExporter

# 1. Set up Trace provider
trace_provider = TracerProvider()
trace_provider.add_span_processor(BatchSpanProcessor(ConsoleSpanExporter()))
trace.set_tracer_provider(trace_provider)

# 2. Set up Log provider
log_provider = LoggerProvider()
log_provider.add_log_record_processor(
    BatchLogRecordProcessor(ConsoleLogRecordExporter())
)
set_logger_provider(log_provider)

# 3. Bridge Python logging into OTel
handler = LoggingHandler(level=logging.INFO, logger_provider=log_provider)
logging.basicConfig(handlers=[handler], level=logging.INFO)
logger = logging.getLogger("exercise-9")

tracer = trace.get_tracer("exercise-9")

# 4. Any log emitted INSIDE a span is automatically correlated
with tracer.start_as_current_span("process-payment") as span:
    span.set_attribute("payment.amount", 49.99)

    logger.info("Payment processing started")     # carries trace_id + span_id!
    logger.warning("Applying discount code")

    with tracer.start_as_current_span("charge-card"):
        logger.info("Charging card")              # span_id = charge-card span

    logger.info("Payment completed successfully")

# The console output for each log record will include:
# "trace_id": "...", "span_id": "...", "trace_flags": 1
# — the same trace_id as the span above!
